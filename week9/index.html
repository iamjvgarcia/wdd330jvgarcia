Client-side Form Validation

  • Web server checks if the data entered by the user is correct.

  • server side validation

    validation done in the server

  • client side validation

    validation done in the browser
    • Two types of client side validation

      • Built-in form validation -

        uses HTML5 form validation features, which we've discussed in many places throughout this module. This validation generally doesn't require much JavaScript. Built-in form validation has better performance than JavaScript, but it is not as customizable as JavaScript validation.

      • JavaScript validation -

        is coded using JavaScript. This validation is completely customizable, but you need to create it all (or use a library).
  • HTML5 APIs
    • Web Storage
      • The Web Storage API provides a key-value store on the clientâ€™s computer that is similar to using cookies but has fewer restrictions, more storage capacity, and is generally easier to use.
    • Geolocation
      • The Geolocation API is used to obtain the geographical position of the device.
    • Web Workers
      • Web workers allow processes to be run in the background, adding support for concurrency in JavaScript.
    • Shared Web Workers
      • These are linked to the script that loaded the worker, and are unable to be used by another script.
    • Service Workers
      • The Service Worker API allows a worker script to run in the background with the added benefit of being able to intercept network requests.
    • Websockets
      • This uses a system of request and response to send data back and forth.
    • Notifications
      • This allows you to show messages using the system's notifications.
    • Multimedia
      • Adding audio and videos
    • A shim or polyfill is a piece of code that adds support of missing features to older browsers.
